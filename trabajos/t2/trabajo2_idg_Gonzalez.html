2.2 Discusi√≥n critica y solucion de la Incompatibilidad Metodol√≥gica

El mayor obst√°culo fue el conflicto de universos Obtenido de la CASEN: Adultos +18 y lo esperado por el CENSO: Poblaci√≥n Total.

  • El Fallo (`NA`s y Weight Populations): El universo de la variable [S2] solo incluye adultos, dejando [NA]s masivos en la muestra (ni√±os y no respondentes). El algoritmo `rakeR` falla si la matriz de control (CENSO) espera 6 millones de personas y la muestra de (Y) no tiene un valor para todos.

  • Soluci√≥n de Ejecuci√≥n: Para garantizar la convergencia (un requisito del entregable) y resolver el conflicto de universos, se opt√≥ por la estrategia de imputaci√≥n a cero sugerida por el profesor precisamente con {S2}-(No respondida o Menor de edad)

Limitaci√≥n Metodol√≥gica: Esta soluci√≥n sesga la tasa estimada hacia abajo, ya que asume que todos los {NA}s (incluyendo 2 millones de ni√±os) son ‚ÄúNo Obesos‚Äù. El mapa resultante es, por lo tanto, una representaci√≥n de la distribuci√≥n relativa del riesgo y no de su magnitud absoluta.

2.3.3 Ejecuci√≥n de la Microsimulaci√≥n (Raking)

Este bloque representa el n√∫cleo del an√°lisis, donde el algoritmo Raking Iterativo (RAKE) transfiere la probabilidad de la variable \(\mathbf{Y}\) (\(\text{Obesidad/Sobrepeso}\)) de la muestra \(\text{CASEN}\) a la estructura poblacional del \(\text{CENSO}\). El proceso se ejecuta a trav√©s de un bucle (lapply) que segmenta la microsimulaci√≥n por comuna para optimizar la convergencia.

El c√≥digo incorpora un mecanismo de programaci√≥n defensiva (tryCatch), que es fundamental dada la limitaci√≥n de la muestra (\(\text{CASEN}\)) y la rigidez de la matriz de control (\(\text{CENSO}\)).

  • Alineaci√≥n de Universos: Antes de la ejecuci√≥n, se utiliza intersect() para asegurar que solo se procesen las comunas que tienen datos v√°lidos en ambos datasets.

  • Gesti√≥n de Fallos: El tryCatch est√° dise√±ado para capturar el error Weight populations don't match (el error estructural que ocurre cuando la matriz \(\text{CASEN}\) es insuficiente para la comuna) y, en lugar de detener el proceso, ignora esa comuna y contin√∫a, garantizando que el script finalice y genere resultados para las comunas viables.

## 3. EJECUCI√ìN DE MICROSIMULACI√ìN (RAKING) ####

cons_censo_comunas = split(cons_censo_df, as.character(cons_censo_df$COMUNA))
inds_list = split(casen, casen$Comuna)
zonas_validas = intersect(names(cons_censo_comunas), names(inds_list))

# Se aplica el proceso de raking solo a las comunas v√°lidas, manejando errores con tryCatch
sim_list = lapply(zonas_validas, function(zona) {
  resultado <- tryCatch({
    cons_i    = cons_censo_comunas[[zona]]
    col_order = sort(setdiff(names(cons_i), c("COMUNA","GEOCODIGO")))
    cons_i    = cons_i[, c("GEOCODIGO", col_order), drop = FALSE]
    tmp     = inds_list[[zona]]
    if (nrow(tmp) == 0) {
      return(NULL)
    }
    
    # 1. Preparaci√≥n de Variables de Control (X)
    inds_i = tmp[, c("ID","edad_cat","esc_cat","sexo_cat"), drop = FALSE]
    names(inds_i) = c("ID","Edad","Escolaridad","Sexo")
    
    # 2. Ponderaci√≥n Iterativa (Raking)
    w_frac  = weight(cons = cons_i, inds = inds_i,
                     vars = c("Edad","Escolaridad","Sexo"))
    
    # 3. Generaci√≥n y Merge Final
    sim_i   = integerise(weights = w_frac, inds = inds_i, seed = 123)
    
    # 4. Asignaci√≥n de la Variable Y (Valor Binario 0/1)
    # NOTA: Usamos el nombre 'tasa_sobrepeso_obeso' que fue imputado y ya no tiene NAs.
    merge(sim_i,
          tmp[, c("ID","tasa_sobrepeso_obeso")], 
          by = "ID", all.x = TRUE)
    
  }, 
  error = function(e) {
    if (grepl("Weight populations don't match", e$message) | grepl("se intenta especificar un atributo en un NULL", e$message)) {
      cat(paste0(">>> ERROR IGNORADO: Fall√≥ la simulaci√≥n para comuna (", zona, "). Contin√∫a...\n"))
      return(NULL)
    } else {
      stop(e)
    }
  })
  
  return(resultado)
})

# Se combinan los resultados de todas las comunas
sim_df = data.table::rbindlist(sim_list, idcol = "COMUNA", fill = TRUE)

2.4.1.1 An√°lisis

El mapa univariado muestra la Tasa Estimada de Sobrepeso u Obesidad (\(\mathbf{S2}\)) en la poblaci√≥n adulta (\(\mathbf{18+}\) a√±os) del Gran Santiago. El an√°lisis de los patrones crom√°ticos permite validar la correlaci√≥n entre la ubicaci√≥n geogr√°fica y el riesgo sanitario.

1.Patr√≥n de Concentraci√≥n del Riesgo

El patr√≥n dominante en el mapa es una clara concentraci√≥n del riesgo sanitario en el anillo externo y perif√©rico de la provincia.
